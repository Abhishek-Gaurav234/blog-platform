üíª Implementation Code

1. Interface Definition

// Enables polymorphism - both proxy and real repo implement this type PostRepositoryInterface interface { FindByID(id int64) (*Post, error) Create(post *Post) error Update(post *Post) error Delete(id int64) error }

2. Proxy Structure

type PostRepositoryCachingProxy struct { realRepository PostRepositoryInterface // The real object cache map[int64]*CacheEntry // In-memory cache maxCacheSize int // LRU limit cacheTTL time.Duration // 5 minutes hits, misses int64 // Statistics }

3. Smart Caching Logic

func (p *PostRepositoryCachingProxy) FindByID(id int64) (*Post, error) { // 1. Check cache first (fast path) if entry, exists := p.cache[id]; exists { p.recordHit() // ‚ö° Cache HIT - instant return return entry.Post, nil } // 2. Cache miss - fetch from database (slow path) p.recordMiss() post, err := p.realRepository.FindByID(id) // üîÑ Database query // 3. Store in cache for next time p.addToCache(id, post) return post, err }

üéØ When to Use Proxy Pattern

‚úÖ Good Use Cases

  • Caching: Reduce expensive database or API calls (our implementation)
  • Lazy Loading: Defer expensive object creation until needed
  • Access Control: Add authentication/authorization checks
  • Logging: Track method calls without modifying original code
  • Remote Proxy: Represent remote objects as local ones

‚ùå When NOT to Use

  • Simple operations that don't benefit from proxying
  • Data that changes very frequently (cache becomes stale)
  • Memory-constrained environments (cache overhead)
  • When adding complexity without clear benefits
